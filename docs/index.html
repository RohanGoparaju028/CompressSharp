01

Why CompressSharp

⚠ The Problem

Large training datasets stored in S3 are often too big to load directly — GPU instances run out of memory, local disks fill up, and data pipelines stall before a single epoch completes.

✓ The Solution

CompressSharp streams the dataset from S3 directly into a GZip compression pipeline, writing only the compressed output to disk. No intermediate full-size download. Compressed files are smaller, faster to read in chunks, and compatible with most ML data loaders.

🔐

Credential Auth

Uses AWS STS GetCallerIdentity to verify your IAM credentials before touching any data.

🔍

Dataset Lookup

Resolves a bucket name and filename against S3 object metadata — fails fast with a clear error if the file does not exist.

🗜️

Stream Compression

Pipes the S3 response stream through GZip without writing the raw file — keeps disk usage to the compressed size only.

Setup & Configuration

CompressSharp reads credentials from a .env file in your working directory. Create the file with the following keys before running:

.env ENV
# Your IAM user credentials (not root) ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY REGION=us-east-1
Required .env Keys
ACCESS_KEY_IDIAM access key ID for the user with S3 read permissions
SECRET_ACCESS_KEYCorresponding IAM secret access key
REGIONAWS region of your S3 bucket, e.g. us-east-1

Install the NuGet dependencies listed in CompressSharp.csproj with dotnet restore before first run.

API Reference

namespace CompressSharp.Auth — Authenticate

Verifies that the credentials in your .env file are valid and have sufficient permissions before any S3 operations begin.

Method Returns Description
Verify() Task<bool> Loads .env, builds credentials, calls STS GetCallerIdentity. Returns true on success, false on any auth failure. Prints the exception message to stdout.
namespace CompressSharp.Search — SearchCsv

Locates a CSV file in S3 by bucket and key, then streams and compresses it to the current working directory.

Member Signature / Type Description
SearchCsv() Constructor Accepts BasicAWSCredentials and RegionEndpoint; instantiates the internal AmazonS3Client.
FindCsv(bucket, filename) Task Checks the S3 object exists via GetObjectMetadata. If found, calls CompressAndDownload. Handles 404 NotFound gracefully with a clear console message.
CompressAndDownload(bucket, filename) Task (internal) Streams the S3 object response directly into a GZipStream. Writes filename.gz to the current directory. No full-size intermediate file is created on disk.
04

Usage Walkthrough

1

Verify credentials

Always call Verify() first. If it returns false, stop — subsequent S3 calls will fail with access denied errors.

2

Instantiate SearchCsv with your credentials

Read the same credentials from your env and pass them to the SearchCsv constructor alongside the region endpoint.

3

Call FindCsv with bucket and filename

Provide the exact S3 key (filename) and bucket name. The library resolves the file, streams it, and writes a .gz file to your working directory.

4

Load the compressed file in your training pipeline

Point your data loader at the .gz file. Most Python ML frameworks (PyTorch, TensorFlow) support reading GZip-compressed CSVs directly via pandas.read_csv(..., compression='gzip').

Program.cs C#
using CompressSharp.Auth; using CompressSharp.Search; using Amazon; using Amazon.Runtime; using dotenv.net; // Step 1 — verify credentials var auth = new Authenticate(); bool ok = await auth.Verify(); if (!ok) { Console.WriteLine("Auth failed. Check your .env credentials."); return; } // Step 2 — build credentials for S3 client var env = DotEnv.Read(); var creds = new BasicAWSCredentials(env["ACCESS_KEY_ID"], env["SECRET_ACCESS_KEY"]); var region = RegionEndpoint.GetBySystemName(env["REGION"]); // Step 3 — locate and compress the dataset var search = new SearchCsv(creds, region); await search.FindCsv("my-ml-datasets", "training/imagenet_labels.csv"); // Output: ./imagenet_labels.csv.gz // Step 4 — use in Python training script: // df = pd.read_csv("imagenet_labels.csv.gz", compression="gzip")
05

Notes & Constraints

📁

CSV files only

The current version targets CSV datasets. Other formats (Parquet, HDF5) are not yet supported.

🌐

IAM permissions needed

Your IAM user needs s3:GetObject and s3:GetObjectMetadata on the target bucket.

📂

Output location

The .gz file is always written to the directory from which the library is invoked — not a configurable path.
